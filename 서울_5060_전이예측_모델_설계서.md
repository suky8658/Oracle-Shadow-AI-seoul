## 1. 프로젝트 개요

### 1.1 문제 정의

서울시 행정동 단위에서 **50~60대 남성 1인가구**를 대상으로, 현재는 고위험군(Q1)이 아닌 행정동이 **향후 Q1으로 전이될 확률**을 사전에 예측하는 AI 모델.

**왜 필요한가?**
- 고독사는 발생 후 사후 대응에 그치는 경우가 많음
- 복지 자원(인력, 예산)은 제한적이므로 사전 집중 배치가 핵심
- "지금 위험한 곳"이 아닌 **"앞으로 위험해질 곳"**을 미리 식별해야 함

### 1.2 핵심 개념

**Dependency Index (편의 의존 지수)**

5개 지표를 가중 합산한 100점 만점 지수. 점수가 높을수록 고독사 위험 패턴에 가까움.

| 가중치 | 지표 | 위험 방향 |
|------|------|--------|
| A: 30% | 편의 인프라 점포수 (log) | 많을수록 ↑ |
| B: 25% | 외출·커뮤니케이션이 모두 적은 집단 비율 | 높을수록 ↑ |
| C: 15% | 배달·식재료 서비스 사용일수 | 많을수록 ↑ |
| D: 15% | 평일 이동횟수 (부호 반전) | 적을수록 ↑ |
| E: 15% | 독거 비율 | 높을수록 ↑ |

**Q1~Q4 사분면 (Quadrant)**

행정동을 Dependency×Avoidance 2축으로 나눈 4개 군집.

```
                Avoidance 높음 (복지 회피)
                        ↑
         Q2 (잠재위험)  |  Q1 (최고위험)
  Dep 낮음 ────────────┼──────────────── Dep 높음
         Q3 (상대안전)  |  Q4 (자기해결)
                        ↓
                Avoidance 낮음 (복지 수용)
```

**전이(Transition)** = Q1이 아닌 행정동이 Q1 기준선(75번째 백분위)을 넘어서는 사건

## 4. 피처 엔지니어링

### 4.1 피처 그룹 (총 ~13개)

| 그룹 | 피처 | 의미 | 시간 범위 |
|------|------|------|--------|
| `level_*` | level_외출커뮤적은, level_이동횟수, level_배달식재료, level_독거비율, level_인프라, level_인구 | 행동 수준의 절대값 | 2022~2023 평균 |
| `delta_*` | delta_외출커뮤적은, delta_이동횟수, delta_배달식재료, delta_독거비율 | 2022→2023 변화량 (조기 악화 신호) | 2022→2023 차분 |
| `infra_*` | infra_slope, infra_delta, infra_recent | 편의 인프라 점포수 장기 추세 | 2022~2025 전분기 |

**왜 2022~2023만 level로 쓰는가?**

레이블(`label_entry`)이 최신연도(2025) 기준이므로, 피처가 레이블과 시간이 겹치면 미래 정보 누출 발생. 조기 구간(2022~2023)만 level 피처로 제한하여 시간적 독립성 확보.

**왜 dep_20XX는 피처에서 제외하는가?**

dep_slope_annual이 레이블 계산(label_slope)에 직접 사용됨 → 모델이 타겟을 미리 보고 학습하는 **데이터 누출** 발생. `dep_20XX`, `dep_slope_annual`, `dep_accel`은 레이블 정의에만 사용하고 피처에서 완전 제외.

### 4.2 build_transition_features.py의 확장 피처 (모듈형에서만)

모듈형 파이프라인은 48개월 월별 시계열을 전부 분석해 더 풍부한 피처를 생성:

| 그룹 | 예시 | 의미 |
|------|------|------|
| `slope_*` | slope_외출커뮤적은 | 48개월 OLS 기울기 |
| `accel_*` | accel_외출커뮤적은 | 후반 slope - 전반 slope (가속도) |
| `vol_*` | vol_외출커뮤적은 | 시계열 표준편차 (불안정성) |
| `delta_*` | delta_외출커뮤적은 | 최근 12개월 평균 - 초기 12개월 평균 |

## 5. Dependency 정규화 방식 — 전역 정규화

### 5.1 문제: 연도별 정규화의 함정

연도별로 따로 minmax 정규화를 하면:
- 2022년 dep=75 와 2025년 dep=75 가 전혀 다른 의미가 됨
- 연도 간 점수 비교 불가 → label_entry(기준연도 vs 최신연도 비교) 자체가 무의미

### 5.2 해결: 전체 연도 풀링 후 단일 정규화

```python
# 모든 연도의 raw 값을 하나로 합친 후 단 한 번만 minmax 적용
pooled["A"] = safe_mm(pooled["_A"].values)   # 전역 min/max 기준
pooled["dep_score"] = (safe_mm(pooled["dep_raw"].values) * 100).round(1)
```

이렇게 하면 "2022년 dep=60 → 2025년 dep=80" 이 실제 악화를 의미함.

## 6. 레이블 정의

레이블은 두 가지 관점에서 정의되며, 모두 **피처와 시간적으로 독립**.

### 6.1 label_slope — 빠른 악화 탐지

```
조건: dep_slope_annual > 0 (악화 방향)
      AND dep_slope_annual >= 양수 slope들의 중앙값

직관: "4년간 Dep 점수가 상승세인 행정동 중 상위 절반"
```

**왜 전체 slope 중앙값이 아닌 양수 slope 중앙값인가?**

전체 중앙값이 음수인 경우, 음수 slope(실제로 개선 중인 행정동)도 레이블=1이 될 수 있음. 양수 slope만 대상으로 중앙값을 구해 "진짜로 악화 중인 상위 그룹"만 선별.

### 6.2 label_entry — Q1 진입 탐지

```
조건: dep_{기준연도} < 75번째 백분위  (처음엔 Q1이 아니었음)
      AND dep_{최신연도} >= 75번째 백분위  (최신 연도에 Q1 진입)

직관: "시작점은 안전했지만 최근에 Q1로 올라온 행정동"
```

75번째 백분위 기준은 전체 연도를 풀링한 **전역 분포**에서 산출하여 연도 간 비교 일관성 유지.

## 7. 모델 설계

### 7.1 모델 파이프라인

```python
# Gradient Boosting (주 모델)
Pipeline([
    ("imp", SimpleImputer(strategy="median")),   # 결측치: 중앙값 대체
    ("clf", GradientBoostingClassifier(
        n_estimators=100,
        max_depth=3,          # 얕은 트리 → 과적합 억제
        learning_rate=0.05,   # 느리게 학습
        min_samples_leaf=10,  # 리프 최소 10개 → 분산 안정
        subsample=0.8,        # 샘플 80%만 사용 → 다양성 확보
        random_state=42,
    )),
])

# Logistic Regression (기준선)
Pipeline([
    ("imp", SimpleImputer(strategy="median")),
    ("sc",  StandardScaler()),
    ("clf", LogisticRegression(
        C=0.1,                     # 강한 정규화
        class_weight="balanced",   # 불균형 레이블 보정
        max_iter=2000,
        random_state=42,
    )),
])
```

**왜 이런 하이퍼파라미터인가?**

N≈419는 ML 모델에 매우 작은 데이터. max_depth=3, min_samples_leaf=10으로 의사결정 경계를 단순화해 과적합을 억제. LR의 C=0.1은 강한 L2 정규화로 동일 목적.

### 7.2 검증 전략

**1차: Stratified 5-fold CV**
- N=419처럼 작은 데이터에서 단일 holdout은 분산이 너무 큼
- 층화 추출(stratify=True)로 각 fold의 양성 비율 유지

**2차: 80/20 홀드아웃 (CV와 독립)**
- CV AUC와 홀드아웃 AUC 차이가 0.05 미만이면 과적합 없음으로 판단
- 샘플이 50개 미만이거나 양성/음성이 10개 미만이면 자동 skip

### 7.3 평가 지표

| 지표 | 의미 | 해석 기준 |
|------|------|--------|
| AUC (ROC) | 전반적 판별력 | 0.5=무작위, 0.7+=유의미, 0.8+=좋음 |
| Average Precision | 불균형 데이터에서의 정밀도-재현율 균형 | 랜덤 기준선 = 양성 비율 |

**왜 Accuracy가 아닌가?**

양성 비율이 약 30~50%이지만 실제 활용 시에는 더 희귀할 수 있음. Accuracy는 다수 클래스에 편향되므로 AUC와 AP를 주 지표로 사용.

### 7.4 위험등급 분류

| 등급 | 전이 확률 범위 | 의미 |
|------|-----------|------|
| 저위험 | 0.00 ~ 0.30 | 현재 추세로 Q1 진입 가능성 낮음 |
| 중위험 | 0.30 ~ 0.50 | 모니터링 필요 |
| 고위험 | 0.50 ~ 0.70 | 선제 개입 검토 |
| 최고위험 | 0.70 ~ 1.00 | 즉시 복지 자원 배치 |

## 8. 자치구 보조 모델

행정동 모델은 Avoidance Index(복지 회피도)를 포함하지 못함(자치구 단위 데이터이기 때문). 보완을 위한 자치구 수준 규칙 기반 모델:

```
전이위험점수 = 0.50 × norm(dep_slope_gu)
             + 0.30 × norm(Avoidance)
             + 0.20 × norm(Dependency)
```

출력: `gu_transition_score.csv`, `gu_transition_map.png`

산점도: Dep 악화 slope(X축) × Avoidance(Y축), 버블 크기 = 전이 위험점수

## 9. 주요 설계 결정 이력 (2026-05-30)

### 9.1 데이터 누출 제거

**문제**: dep_slope_annual이 label_slope 정의에 직접 사용됨 → 모델이 타겟을 미리 봄

**조치**:
- `FEAT_COLS`에서 `dep_20XX`, `dep_slope_annual`, `dep_accel` 제거
- `save_model.py`의 `FEAT_PREFIXES`를 `["level_", "delta_", "infra_"]`로 한정
- `train_transition_model.py`의 `load_data()`에서 dep_cols 추가 코드 삭제

### 9.2 전역 정규화로 교체

**문제**: 연도별 minmax → 연도 간 점수 비교 불가

**조치**: 4개 연도 전체를 풀링한 뒤 단일 `safe_mm()` 한 번 적용

### 9.3 label_slope 조건 개선

**문제**: 전체 slope 중앙값이 음수일 경우, 감소 중인 행정동도 레이블=1

**조치**: `양수 slope들의 중앙값`을 기준으로 변경

### 9.4 label_entry 전역 기준 통일

**문제**: 기준연도에서 연도별 75번째 백분위를 쓰면 기준 자체가 달라짐

**조치**: 전체 연도 풀링 분포에서 단일 75번째 백분위(전역) 계산 후 사용

### 9.5 홀드아웃 검증 추가

CV만으로는 과적합 여부 판단 어려움 → 80/20 무작위 분할 홀드아웃 병행

### 9.6 OLS slope 최소 관측치 완화

`len(s) < 3` → `len(s) < 2`로 변경: 관측치 2개인 행정동도 slope 계산 가능

- [ ] **생존 분석(Survival Analysis)**: "언제" Q1으로 전이하는지 예측
