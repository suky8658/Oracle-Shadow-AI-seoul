**SHADOW** = **S**eoul **H**idden **A**voidance & **D**ependency **O**verlap **W**atch

## 1. 문제 정의

| 단계 | 확정 사실 | 다음 질문 |
|---|---|---|
| ① 누구에게 | 2024년 전체 고독사의 **54.0%가 50·60대 남성**. | 어느 지역에서 두드러지나? |
| ② 왜 서울 | 인구당 발생률은 중상위권이나, **전체 사망 대비 고독사 비중은 전국 1위**. | 생활 조건은? |
| ③ 취약 집중 | 단순노무·이혼/사별 등 복합 취약성이 특정 자치구에 집중(생계급여 밀도와 r=0.58). | 도시 인프라는 고립을 완화하나, 은폐하나? |
| ④ 제도의 공백 | 남성 고독사의 **62.2%가 복지 미수혜**. 돌봄이 닿은 경우도 **78.9%가 공식 채널**. | 가장 잘 찾는 도시에서, 무엇이 닿지 못했나? |

→ EDA의 종착점은 **두 개의 역설**이다.

## 2. 두 개의 역설

고립 위험을 하나의 점수로 합치지 않고 **두 축으로 분리**해 진단한다.

- **편의의 역설 (무의식적 고립)**: 편의 인프라가 혼자 살아도 불편하지 않게 만들어 외출·대면 접촉의 필요를 줄이고, 고립을 본인도 사회도 모르게 고착시킨다.
- **복지의 역설 (의식적 고립)**: 복지와 발굴 체계가 있어도 낙인·거부감·남성성 규범 때문에 실제 이용으로 이어지지 않는다.

두 역설은 원인도 작동 방식도 달라, 통계적으로 거의 독립된 별개의 축으로 확인된다.

## 4. SHADOW Map - 진단

두 축으로 서울 25개 자치구를 4개 고립 유형으로 분류한다.

**Dependency Index (편의 의존)**, 5요소 가중합 (100점)

| 가중치 | 요소 |
|---|---|
| 0.30 | 편의 인프라 밀도 (점포수, log) |
| 0.25 | 외출·커뮤니케이션 모두 적은 비율 |
| 0.15 | 배달·식재료 사용일수 |
| 0.15 | 평일 이동횟수 (부호 반전) |
| 0.15 | 독거 비율 |

> 검증: 편의 인프라 ↔ 외출·소통 부재 r=0.648, 1인가구 비율 통제 후에도 부분상관 0.596 유지.

**Avoidance Index (복지 회피)**, 4요소 (서울서베이)

| 요소 | 의미 |
|---|---|
| 도움부재 | 도움받을 사람 없음 |
| 외로움부정 | 외로움·도움 필요성을 낮게 인정 (부정 갭) |
| 복지불신 | 제도에 대한 불신 |
| 네트워크축소 | 관계망 축소 |

> 복지시설 밀도 ↔ Avoidance 무상관(rho=0.099). 회피는 **시설 부족이 아니라 낙인 문제**.

| 유형 | 상태 | 처방 방향 |
|---|---|---|
| **Q1** 최고위험 | 의존↑ · 회피↑ | 낙인 낮춘 비대면 선제 개입 |
| **Q2** 정서취약 | 의존↑ · 회피↓ | 기존 복지 사업 직접 연계 |
| **Q3** 상대안전 | 의존↓ · 회피↓ | 경량 모니터링 |
| **Q4** 자기해결 | 의존↓ · 회피↑ | 복지 언어 배제, 상권·커뮤니티 기반 간접 발굴 |

## 5. SHADOW AI - 예측

현재는 안전하지만 **곧 Q1으로 전이될 행정동**을 선제 예측한다. 같은 점수라도 '내려가는 중'과 '빠르게 올라가는 중'은 위험이 다르므로, 4년치 시계열의 수준·추세·변화량을 피처로 학습한다.

> **데이터 누출 방지:** 레이블 정의에 쓰인 연도별 Dependency 점수와 그 기울기는 피처에서 제외하고, 조기 구간 변수만 사용한다.

**모델 & 성능**: Gradient Boosting (주) + Logistic Regression (기준선). N≈419로 작아 얕은 트리·강한 정규화로 과적합을 억제하고, Stratified 5-fold CV에 80/20 홀드아웃을 더해 이중 검증.

| 레이블 | 모델 | AUC | AP |
|---|---|---:|---:|
| label_entry | **Gradient Boosting** | **0.844** | 0.302 |
| label_entry | Logistic Regression | 0.842 | 0.271 |

행정동별 위험 동인은 **SHAP**으로 분해한다. 예측값은 개인이 아닌 **지역 단위 신호**다.

## 6. SHADOW RAG - 처방

진단·예측을 받아 **"이 지역에 어떤 제도가 왜 필요한가"**를 근거와 함께 제안한다. 기존 제도를 정답이 아니라 **한계를 가진 기준선**으로 본다.

```
① 현행 제도     "이런 제도가 이미 있습니다"
② 한계 진단     "낙인 유발 / 의존 심화 / 자치구 귀속 / 중복으로 부족합니다"
③ 필요 방향     "그래서 ~한 방향의 제도가 필요합니다"
④ 외부 레퍼런스 "그 방향은 해외·타지역에서 이미 검증됐습니다"
⑤ 이식 제안     "이 지역에도 이런 제도가 마련되어야 합니다"
```

**뉴로-심볼릭 (무환각):**
- **추론(결정적):** 전문가가 설계한 복지 온톨로지(지식그래프)를 탐색해 한계를 논리적으로 도출한다. 사분면이 민감해하는 요인과 제도가 자극하는 요인이 겹치면 충돌로 본다.
- **생성(LLM):** LLM은 도출된 근거 **안에서만** 처방을 작성한다(근거 밖 생성 금지, 출처 필수).
- **벡터/임베딩 미사용**: 지식이 구조적이라 그래프가 더 정확하고 설명 가능하다.

→ **전문가 온톨로지로 한계를 결정적으로 진단하고, LLM이 외부 검증 제도를 이식 처방으로 생성하는 무환각·설명가능 시스템.**

## 7. 폴더 구조

```text
Oracle-Shadow-AI-seoul/
├─ shadow_service.py        # 대시보드 진입
├─ shadow_rag_llm.py        # 처방 엔진 (ADB 그래프 쿼리 + LLM 생성)
├─ shadow_chat.py           # 담당자 질의응답 챗봇
├─ rag_graph.py             # 처방 지식그래프 시각화
│
├─ code/                    # 분석: 지수·전이예측 산출
├─ Outputs/                 # 산출물 (지수·예측 CSV, 모델, 리포트)
├─ Data/                    # 통합 처방 인풋 (shadow_profile.json)
├─ pipeline/                # 데이터 통합 (build·merge·verify)
├─ views/ · kb/             # 대시보드 부품
└─ deploy/                  # OCI 클라우드 배포
```

## 8. 실행

```bash
pip install -r requirements.txt

# 대시보드 (진단맵 · 전이예측 · RAG처방)
python -m streamlit run shadow_service.py
```

대시보드는 OCI Compute 가상머신에 배포해 공개 웹서비스로 운영했으며, 처방 지식그래프는 OCI Autonomous Database에서 동작한다.

## 9. 한계와 향후 방향

| 한계 | 완화/계획 |
|---|---|
| 소표본 (N≈419) | 얕은 트리·강한 정규화로 과적합 억제 |
| 생태학적 오류 (집계 단위) | 개인 데이터 2단계 모델 (중기) |
| Avoidance 행정동 미포함 | 자치구 보조 모델로 보완 |
| Ground Truth 부재 | 실제 고독사 발생 데이터로 외부 검증 (중기) |

**향후:** 행정 데이터 연계 자동 갱신 파이프라인 · 대상/지역 확장(전국·타 취약군).
