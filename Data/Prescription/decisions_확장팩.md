# 판단 로그 — 서울 외 도시 + 해외 통합 패키지 (532건)

> `AGENT_PROMPT_programs_enrich.md` §4½ rule9에 따라, `tagging_guide`에 없거나 애매한 케이스의 판정 기준과 이유를 기록.
> 병합 시 팀원(이여진·고수경) 로그와 비교해 기준 통일할 것.
> 한 배치 전체에 **일관 적용**(플립플롭 금지)했음.

## 0. 패키지 구성·담당 (532건)

- **경기 GG 368건**: 팀원 작업분(`gyeonggi_*`). 별도 decisions 로그는 없으나 `gyeonggi_audit.json`에 코드별 근거+rule 참조 내장(예: "rule2 a"). 본 로그의 A~H 기준과 동일 `tagging_guide`를 따름. ⚠️ access_mode가 보편 365/368로 쏠림 → 병합 검수 시 P의 선별 우세 분포와 기준 차이 있는지 교차확인 권장.
- **부산·전라·제주 P 50건 + 해외 O 114건**: 본인 작업분. 아래 A~H가 그 판정 기준.

## A. 전달방식 → 의존 코드 (rule1/rule8 적용·확장)

- **A1. 경로식당 무료급식(집합형)은 DEP_NoOuting 미부여.** 식당에 모여 먹으면 외출·접촉이 유지되므로 NoOuting 안 줌. 상시 급식 전제만 DEP_FragileSupply 부여.
  - 적용: P-0001·P-0002(경로식당+배달 혼합 → 배달분 때문에 NoOuting 부여), P-0021·P-0035(순수 경로식당 → NoOuting 미부여, FragileSupply만)
- **A2. 식사·물품 배달형(밑반찬·도시락·음료·우유)은 DEP_NoOuting + DEP_FragileSupply.** 집으로 배달돼 외출 동기 소멸 + 정기 공급 전제.
  - 적용: P-0019·P-0029·P-0036(식사배달), P-0009·P-0012·P-0033(음료/간편식 안부배달)
- **A3. 방문형 안부살피기는 DEP_NoContact 미부여.** 사람이 방문(대면 유지)하므로 NoContact 아님. 정기 방문 전제만 FragileSupply.
  - 적용: P-0027(노인맞춤돌봄 방문 음료)
- **A4. 비대면 IoT/센서/AI 모니터링은 DEP_NoContact + DEP_NoOuting + DEP_FragileSupply.** 사람 접촉 없이 기기로 완결 + 상시 관제.
  - 적용: P-0010(TV안부알리미), P-0014(센서돌봄), 제주 P-0048(빅데이터)·P-0051(응급안전)·P-0052(AI스피커)

## B. 의료보조·일회성 현물 → 의존 미부여/NoRelation (rule6 적용)

- **B1. 의료보조기구(보청기·보행기) 일회성 지급은 DEP 미부여.** 이동·청력을 보완해 외출·사회참여를 오히려 돕는 enabling이므로 의존 남발 금지(rule6). 낙인(WelfareLabel+SelfAction)만.
  - 적용: P-0005·P-0020·P-0026(보청기), P-0031(보행기)
- **B2. 일회성 현물(생신상·명절선물·장례용품)은 DEP_NoRelation.** 단발 지급이라 관계망 미형성. 반복·상시 아니므로 FragileSupply 미부여.
  - 적용: P-0006·P-0007·P-0016(생신상/선물), P-0030(장례용품)
- **B3. 소득선별 일회성 현금(냉난방비·임차료)은 DEP 미부여.** rule6대로 현금성 일회성은 의존 남발 금지.
  - 적용: 제주 P-0050(에너지드림), P-0054(주거비)

## C. fulfills_needs 부여 기준 (rule2·rule3)

- **C1. 노인 일자리·사회참여는 NEED_NoStigma + NEED_RelationRestore.** '복지 수혜'가 아닌 활동 주체 형식 + 관계활동.
  - 적용: P-0023(골목호랑이 어르신단)
- **C2. 소득무관 보편 위생(목욕·이미용)은 NEED_NoStigma.** 자격심사 없이 연령만으로 보편 지급 + 생활위생 언어.
  - 적용: P-0022·P-0042
- **C3. 취업·자립역량 지원은 NEED_RelationRestore (rule2-b 이전가능 생활자립역량).** 단 고립·안부와는 결이 달라 off-theme 표시.
  - 적용: P-0015·P-0041(경력단절 자립)
- **C4. 병원동행은 NEED_RelationRestore.** 사람이 동행해 외출·인적 접촉을 동반(E-015 동행모델 유사). 단순 대행 아님.
  - 적용: P-0046
- **C5. 발굴형 생활밀착(무낙인)은 NEED_NoStigma.** 본인 신청·복지라벨 없이 생활/우편 인프라로 발굴.
  - 적용: P-0011(이웃연결상점), P-0013(집배원 안부살핌)

## D. access_mode = 중복배제 판정

- **D1. "유사·타 사업과 중복 수혜 불가" 명시 시 중복배제.** 단순 소득선별과 구분.
  - 적용: P-0029(장흥 식사배달 '경로식당·돌봄 중복 불가'), 제주 P-0050(에너지바우처 제외)·P-0052(응급안전 제외)·P-0053(장기요양·가사간병 제외)

## E. target_age 추정 (rule7)

- **E1. '고독사 위험 1인가구'·'고립 위험 1인가구'로 연령 미명시 시 → [중장년, 노인] 추정.** _audit에 '원문 연령 미명시' 명기.
  - 적용: P-0012·제주 P-0048·P-0049(50~65 명시는 중장년 단독)
- **E2. enum 외 생애주기(아동·청소년·영유아)는 enum 해당분만 채택.** 예: 아동·청소년·청년·중장년 → [청년, 중장년].
  - 적용: P-0003·P-0034(장애인활동지원)
- **E3. 영유아~노년 전 생애주기 → [전연령].**
  - 적용: P-0008(찾아가는 복지한마당)

## F. off-target 플래그 기준

- **F1. 수혜 대상이 1인가구/고립 노인이 아니면 off-target.** 종사자수당(대상=종사자)·다문화 모국방문·결혼이민자·재난구호·장례는 고립·안부 결 아님.
  - 처리: 사용자 지시로 종사자수당(P-0017·28)·다문화(P-0037·40·43)·장례(P-0030) **6건 삭제**. 경력단절(P-0015·41)·재난구호(P-0024)는 유지하되 `_meta.scope` 표시.

## G. 해외 이식후보(O) 고유 판정

- **G1. 해외는 '좋은 설계를 이식'하므로 stimulates_stigma·deepens_dependency 비움.** fulfills_needs(NoStigma/RelationRestore)만 부여 = transplant_rule 매칭용.
- **G2. evidence.json에 근거로 있던 유명정책 16건(O-0049~64)은 evidence_ids로 E-###에 교차연결.** 같은 정책이 근거(evidence)+이식후보(programs) 양 레이어에 존재(용도 다름).
- **G3. 데이터발굴 반면교사(O-0074~76: SyRI·글라드삭세·앨러게니)도 포함.** 인권침해·편향으로 폐지·논란된 사례지만 우리 AI 발굴 설계의 윤리적 경계 자료로 가치. `_meta.유형='데이터 발굴(반면교사)'` + `note`에 폐지 사유 명기.
- **G4. 출처 신뢰도**: 1차(정부·기관 공식) 우선. 차단·미확인 시 교차확인 출처로 대체하고 `_meta.출처종류`에 표기(예: Men's Pie Club 공식차단→스니펫, 라포스트→WEF). 작동 URL 없으면 폐기.

## H. 데이터 수집상 함정(메모)

- **H1. 지자체복지서비스 API에 '1인가구' 정식 가구상황 코드 없음.** → 1인가구는 이름·본문 텍스트매칭으로만 식별(지역당 얇음). 중장년·노년은 생애주기 정식코드라 신뢰.
- **H2. API 응답 EUC-KR 아님 UTF-8**, 요청 파라미터 UTF-8 인코딩, `type=json` 붙이면 403(XML만). 일일 트래픽 1000 한도 → resume 설계.
